# Substrate

Substrate is an event-backed state-object store. This repository contains the v0 foundation: flexible workspace-scoped collections, record CRUD with optimistic concurrency and idempotency, and full time-travel — per-record history, point-in-time reads (`as_of`), and revert to any past revision.

---

## Prerequisites

| Tool | Version |
|------|---------|
| [Go](https://go.dev/dl/) | 1.26+ |
| [Task](https://taskfile.dev) | any recent |
| [Docker](https://docs.docker.com/get-docker/) | for integration tests |

> This project uses a `Taskfile.yml` (not a Makefile). Install Task with `brew install go-task` or see https://taskfile.dev/installation/.

## Build & test

```sh
# Build all packages
task build
# equivalent: go build ./...

# Unit tests (no external dependencies)
task test
# equivalent: go test ./...

# Integration tests (requires Docker — spins up Postgres via embedded-postgres)
task test:integration
# equivalent: go test -tags=integration ./...

# Regenerate the sqlc db package after editing SQL queries or schema
task sqlc:generate
```

**Data layer:** database migrations are managed by [goose](https://github.com/pressly/goose) (embedded, applied automatically at startup). Type-safe queries are generated by [sqlc](https://sqlc.dev) with the [pgx](https://github.com/jackc/pgx) driver and live in the `db` package.

## Running the server

```sh
# Quickest start — embedded Postgres (no external DB needed)
task run
# equivalent:
go run ./cmd/substrate --embedded --admin-token mysecret

# External Postgres
go run ./cmd/substrate --database-url "postgres://user:pass@localhost:5432/substrate?sslmode=disable" \
                       --admin-token mysecret
```

### Flags

| Flag | Env var | Default | Description |
|------|---------|---------|-------------|
| `--addr` | — | `:8080` | Listen address |
| `--embedded` | `SUBSTRATE_EMBEDDED=true` | `false` | Start an embedded Postgres instance |
| `--database-url` | `SUBSTRATE_DATABASE_URL` | — | External Postgres DSN |
| `--admin-token` | `SUBSTRATE_ADMIN_TOKEN` | — | Bootstrap admin token |
| `--log-level` | — | `info` | Log level |

## Admin bootstrap

Create a workspace and issue an API key. The `key` field in the response is the plaintext secret — store it now, it is not retrievable later.

```sh
export ADMIN_TOKEN=mysecret
export BASE=http://localhost:8080

# 1. Create a workspace
WS=$(curl -sf -X POST "$BASE/admin/workspaces" \
  -H "X-Admin-Token: $ADMIN_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"name":"acme"}' | jq -r '.id')

# 2. Issue an API key
KEY=$(curl -sf -X POST "$BASE/admin/workspaces/$WS/api-keys" \
  -H "X-Admin-Token: $ADMIN_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"label":"dev"}' | jq -r '.key')
```

## API walkthrough

All data-plane routes are under `/v1/`. Authenticate with `Authorization: Bearer <key>`. Use `X-Substrate-Actor` to tag the actor on each write (optional; defaults to `"anonymous"`).

```sh
export BASE=http://localhost:8080
export AUTH="Authorization: Bearer $KEY"
export ACTOR="X-Substrate-Actor: alice"

# Create a collection
curl -sf -X POST "$BASE/v1/collections" \
  -H "$AUTH" -H "$ACTOR" \
  -H "Content-Type: application/json" \
  -d '{"name":"products"}'

# Create a record (Idempotency-Key makes the call safe to retry)
REC=$(curl -sf -X POST "$BASE/v1/collections/products/records" \
  -H "$AUTH" -H "$ACTOR" \
  -H "Content-Type: application/json" \
  -H "Idempotency-Key: create-widget-001" \
  -d '{"data":{"sku":"WIDGET-001","price":9.99}}')
ID=$(echo "$REC" | jq -r '.id')
ETAG=$(echo "$REC" | jq -r '.revision')

# Read the record (response includes ETag header with the current revision)
curl -sf "$BASE/v1/collections/products/records/$ID" \
  -H "$AUTH"

# Update with optimistic concurrency (If-Match must match current revision)
curl -sf -X PATCH "$BASE/v1/collections/products/records/$ID" \
  -H "$AUTH" -H "$ACTOR" \
  -H "Content-Type: application/json" \
  -H "If-Match: \"$ETAG\"" \
  -H "Idempotency-Key: update-widget-001-price" \
  -d '{"data":{"sku":"WIDGET-001","price":12.99}}'

# Point-in-time read (as_of accepts revision number, event UUID, or RFC3339 timestamp)
curl -sf "$BASE/v1/collections/products/records/$ID?as_of=1" \
  -H "$AUTH"

# Full revision history
curl -sf "$BASE/v1/collections/products/records/$ID/history" \
  -H "$AUTH"

# Revert to a past state (to accepts same formats as as_of)
curl -sf -X POST "$BASE/v1/collections/products/records/$ID/revert" \
  -H "$AUTH" -H "$ACTOR" \
  -H "Content-Type: application/json" \
  -d '{"to":"1"}'

# Delete
curl -sf -X DELETE "$BASE/v1/collections/products/records/$ID" \
  -H "$AUTH" -H "$ACTOR"
```
